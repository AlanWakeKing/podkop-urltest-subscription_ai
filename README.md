# Podkop Auto Config Script
Это вспомогательный инструмент для OpenWrt, который добавляет недостающую функциональность работы с подписками в плагин Podkop (sing-box).

По умолчанию Podkop не имеет встроенной системы для автоматической загрузки и обновления ключей из внешних ссылок-подписок. Данный скрипт устраняет это ограничение, превращая статическую конфигурацию в динамическую систему с автоматическим перебором серверов.

## 🚀 Возможности
- 📥 Поддержка RAW ссылок (GitHub и др.)
- 🌐 Парсинг HTML/текстовых страниц с ключами
- 🔗 Поддержка прямых ссылок (vless://, ss://, trojan:// и др.)
- ⭐ Приоритизация прямых ключей (всегда попадают в конфиг)
- 🎲 Случайная выборка серверов до установленного лимита
- 🧹 Очистка и нормализация конфигов
- ❌ Фильтрация невалидных параметров (xhttp, пустой security и т.д.)
- 🛡 Защита от поломки конфига (fallback)
- ⚡ **Автоматическая интеграция с Podkop (URLTest, Cron)**

## ⚙️ Инструкция по установке

### ✦ Шаг 1: Установка
Подключитесь по **SSH** к роутеру и выполните команду:
```bash
wget -O /usr/bin/podkop-update https://raw.githubusercontent.com/Naz1anmak/podkop-urltest-subscription/main/update.sh && \
chmod +x /usr/bin/podkop-update
```

### ✦ Шаг 2: Настройка источников
Откройте файл в **vi** или **nano** и впишите свои ссылки в переменные `SOURCE_1`, `SOURCE_2`, `SOURCE_3`:

```bash
vi /usr/bin/podkop-update
```
(нажмите i для редактирования, Esc затем :wq для сохранения и выхода).

Или установи более удобный редактор (apk add nano)
```bash
nano /usr/bin/podkop-update
```
Сохраните изменения и выйдите из редактора.

Если провайдер подписки требует device id в HTTP-заголовке, скрипт может сам создать постоянный `HWID` и сохранить его в `/etc/podkop-hwid`.

По умолчанию ничего вручную генерировать не нужно: при первом запуске `podkop-update` создаст UUID и дальше будет использовать один и тот же идентификатор.

Если нужен ручной override, заполните переменную `X_HWID`:
```bash
X_HWID="ваш-hwid"
```
При желании можно изменить и путь хранения:
```bash
HWID_FILE="/etc/podkop-hwid"
```
Скрипт будет добавлять заголовок `X-HWID` только при загрузке подписок. Для прямых ссылок `vless://`, `ss://`, `trojan://` и других локальных ключей он не используется.

Если вы хотите, чтобы ваш backend видел устройство подробнее, можно также задать:
```bash
SUBSCRIPTION_USER_AGENT="Happ/2.7.0/Windows/2604031533507"
X_DEVICE_OS="Windows"
X_VER_OS="10.0.19045"
X_DEVICE_MODEL="Parts_x86_64"
```
Тогда при запросе подписки скрипт отправит:
- `User-Agent: ...`
- `X-HWID: ...`
- `X-Device-OS: ...`
- `X-Ver-OS: ...`
- `X-Device-Model: ...`

Для сервиса `bot-vpn` это важно: он читает именно `X-Device-OS`, `X-Ver-OS` и `X-Device-Model`. Старые `X-OS` и `X-Model` оставлены только для совместимости.

Для OpenWrt можно оставить значения по умолчанию:
```bash
X_DEVICE_OS="OpenWrt"
X_VER_OS=""
X_DEVICE_MODEL=""
```
Тогда версия OpenWrt и модель роутера будут по возможности определены автоматически.

`IP адрес`, `Последний доступ` и `Количество обращений` формируются уже на стороне вашего сервиса по самому факту HTTP-запроса.

### Проверка вручную
После настройки запустите скрипт вручную:
```bash
/usr/bin/podkop-update
```
Если всё прошло без ошибок, Podkop будет обновлён.

> [!IMPORTANT]
> В админке могут быть ошибки, но Podkop с ними запустится.

### ✦ Шаг 3: Автообновление (Cron)
По умолчанию скрипт сам добавляет и поддерживает cron-задачу:
```cron
0 */3 * * * /usr/bin/podkop-update >> /var/log/podkop-update.log 2>&1
```
То есть обновление будет запускаться каждые 3 часа.

Если хотите изменить расписание, отредактируйте переменную:
```bash
CRON_SCHEDULE="0 */3 * * *"
```

Если не хотите, чтобы скрипт автоматически трогал `cron`, отключите:
```bash
AUTO_INSTALL_CRON="0"
```

Лог выполнения будет сохраняться в:
`/var/log/podkop-update.log`

## 🧹  Скрипт удаления
Удалить скрипт и задание cron можно одной командой:
```sh
rm -f /usr/bin/podkop-update && \
sed -i '/podkop-update/d' /etc/crontabs/root && \
/etc/init.d/cron restart
```

## 🧠 Как это работает
- Загружает конфиги из источников
- Извлекает ключи (vless, ss, trojan и др.)
- Нормализует параметры
- Удаляет невалидные (например xhttp)
- Добавляет прямые ключи в приоритет
- Случайно добирает до лимита
- Применяет конфиг в Podkop

## ⚠️ Требования
- Доступ к интернету
- Podkop установлен
> [!NOTE]
> Могут быть ошибки из-за различий названий полей, скрипт для версии 0.7.14

## 📌 Примечания
- Проверка доступности серверов выполняется самим Podkop (URLTest)
- Если ключей нет или после фильтрации не осталось рабочих вариантов, текущий конфиг не меняется
